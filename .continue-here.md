## What Was Being Done

用户希望将看板通过花生壳内网穿透映射到公网，让任何人打开网页都能看到数据，而无法获取API key。

**目标架构：**
- 所有外部API请求通过Python后端代理
- API key只存储在后端环境变量
- 前端JavaScript不包含任何API key
- nginx只代理到Python后端

## Key Decisions Made

1. **纯后端代理架构**：所有需要API key的请求都通过后端，前端零key
2. **缓存策略**：
   - FRED: 5分钟TTL
   - BLS: 30分钟TTL（配额25次/天）
   - Alpha Vantage: 1小时TTL（配额25次/天）
   - Binance: 30秒TTL（实时数据）
3. **nginx简化**：删除所有直接外部API代理，只保留backend代理
