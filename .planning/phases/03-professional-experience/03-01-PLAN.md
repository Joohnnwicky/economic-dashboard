| Binance WebSocket → Client | Public streaming endpoint, untrusted input (malformed JSON possible) |
| Client → TanStack Query cache | Internal state update, trusted within application |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | WebSocket message handler | mitigate | Parse JSON directly per D-05 (performance), but validate price is numeric before setQueryData |
| T-03-02 | Denial of Service | WebSocket connection | mitigate | Exponential backoff + max retry limit (D-16) prevents infinite reconnection attempts |
| T-03-03 | Denial of Service | Heartbeat | accept | 30s ping is minimal overhead, prevents server-side disconnect |
| T-03-04 | Information Disclosure | API keys | accept | Binance public streams require no authentication, no secrets exposed |
</threat_model>
